# color → 90% real-world precision — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise `tokens/no-hardcoded-color` to ≥ 0.90 precision and ≥ 0.90 recall on a labelled sample of its findings on real OSS code, reproducible in-repo; promote it into the score (dedicated v2→v3 bump) iff it clears the gate.

**Architecture:** Harvest real findings from cloned public OSS design systems → label them against the UI-styling drift rubric (human adjudicates the ambiguous tail) → for each real false-positive class, add a narrow AST guard in `_skip-context.ts` (recall-preserving) → vendor the labelled snippets as in-repo fixtures → measure via sub-project A's `deriveMeasurement` + catalogue-coherence test → conditionally promote.

**Tech Stack:** TypeScript (strict), vitest, tsx, the audit pipeline (`auditDirectory`), sub-project A's reliability infra (`deriveMeasurement`, `falseFriends`, coherence test, `shouldPromote`).

## Global Constraints

- Strict TS: `exactOptionalPropertyTypes`, `noUncheckedIndexedAccess`, `verbatimModuleSyntax`; ESM `.js` specifiers.
- Determinism byte-for-byte; no `Date.now()`/`Math.random()`; `lastCalibrated` a fixed string.
- Cloned repos are git-ignored, NEVER committed. Only distilled labelled snippets are committed (with source attribution where the license requires).
- Rubric (verbatim): **TP** = color literal in UI-styling code (CSS, SCSS, CSS-in-JS, inline `style`, arbitrary `className`). **FP** = token-definition files, stories/tests/examples/fixtures, data/chart/syntax palettes, SVG/icon art, config/data. Keywords (`transparent`/`currentColor`/`inherit`) and `var()` fallbacks are not drift.
- Anti-bias: labels come from the real code context, NEVER from "what makes the rule pass." Recall non-regression gate on every guard change.
- Score changes ONLY if the gate (precision ≥ 0.90 ∧ recall ≥ 0.90 ∧ N ≥ 30) is cleared, and only via a `CURRENT_SCORING_VERSION` bump + a new `LOCKED` entry in `scoring-contract.test.ts` (never edit the existing entry).
- Conventional Commits; branch `feat/color-to-90` (stacked on `feat/socle-mesure-honnete`). Trailers: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>` and `Claude-Session: https://claude.ai/code/session_01MoiMorjpU4r6NaYfctP96w`.
- All artifacts in English.
