# ai-governance/ai-marker-anti-patterns

**Axis:** `ai-governance` | **Severity:** `warning` | **Status:** experimental

Lint two forbidden AI-marking anti-patterns in component source files.

---

## Why this rule exists

Two established design system guidelines explicitly forbid specific AI-marking patterns that look plausible but break accessibility and UX clarity:

**Anti-pattern A — sparkle as the sole AI marker.**
A sparkle icon (✨, `Sparkles`, `SparkleIcon`) used without any accompanying text label or AI-marker component tells sighted users "something AI-related is here" but conveys nothing to users who rely on assistive technology. SAP Fiori XAI requires that sparkle be paired with a text label or a dedicated AI-marker component. A lone sparkle also fails at the semantic level: it has no stable, agreed-upon meaning across design systems.

**Anti-pattern B — "AI" as a CTA action label.**
Labelling a button or link "AI" (or "Ask AI") puts the technology in the subject position, not the action. Users want to know what will happen ("Summarize", "Explain", "Generate"), not what engine is doing it. GitLab Pajamas explicitly forbids `AI` as the primary action label of a CTA element. The word is a noun, not a verb.

---

## How it works

The rule globs `**/*.{tsx,jsx,vue}` (excluding `node_modules/`, `dist/`, `build/`) and runs two independent per-file detectors on the raw source text.

**Detector A** fires when:
- A sparkle signal is present: `✨` / `✩` / `🌟` / `❇` literals, or import/use of `Sparkle`, `Sparkles`, `SparkleIcon`, or an `icon="sparkle*"` prop.
- AND no text AI-marker accompanies it anywhere in the same file: no `Generated by AI`, `AI-generated`, `Powered by AI`, no standalone `\bAI\b`, and no recognised AI-marker component tag (`AILabel`, `AIBadge`, `magic-*`, `GenAI*`, etc.).

**Detector B** fires when:
- A `<button>`, `<Button>`, or `<a>` element contains the standalone, case-sensitive token `AI` as (part of) its visible label text.
- "Standalone" means `AI` is not a substring of a longer word (`Train`, `Maintain`, `detail` do not trigger). The check is case-sensitive — lowercase `ai` does not trigger.

Findings are sorted by file path then line number for deterministic output. Both detectors emit `warning` severity.

## Examples

### Anti-pattern A — sparkle without text marker

**Bad — sparkle only, no label:**

```tsx
export function Summary() {
  return <span>✨ {generatedText}</span>;
}
```

**Good — sparkle paired with AI-marker component:**

```tsx
import { AILabel } from "@acme/ds";

export function Summary() {
  return (
    <span>
      <AILabel>AI</AILabel>
      <Sparkles aria-hidden />
      {generatedText}
    </span>
  );
}
```

**Good — sparkle paired with visible text marker:**

```tsx
export function Summary() {
  return <span>✨ AI-generated summary: {generatedText}</span>;
}
```

### Anti-pattern B — "AI" in a CTA label

**Bad — "AI" as the action label:**

```tsx
<button>AI</button>
<button>Ask AI</button>
<Button>AI</Button>
```

**Good — describe the action, not the technology:**

```tsx
<button>Summarize</button>
<button>Explain this</button>
<button>Generate draft</button>
```

---

## Auto-fix

No automatic fix is available. Fixing anti-pattern A requires choosing an AI-marker component from your design system (or adding a visible text label). Fixing anti-pattern B requires agreeing on the action verb with your product/content team — this is a product decision.

---

## Allowlist

To disable this rule for an entire repository, add the following directive to a `README.md`, `README`, `README.mdx`, `readme.md`, `.lyse.yaml`, or `.lyse.yml` at the repository root:

```
lyse-disable ai-governance/ai-marker-anti-patterns
```

When the directive is present, the rule returns zero findings and `opportunities: 0`.

Additional items that are never flagged:
- A sparkle accompanied by a text AI-marker or an AI-marker component (`AILabel`, `AIBadge`, `magic-*`, `GenAI*`, `*AIMarker*`).
- `AI` appearing only in non-CTA elements (headings, paragraphs, spans, `<h2>AI suggestions</h2>`).
- Files larger than 1 MB.
- Files under `node_modules/`, `dist/`, `build/`.
- Lowercase `ai` in button labels (case-sensitive check).
