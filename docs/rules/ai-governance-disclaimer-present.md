# ai-governance/disclaimer-present

## Why

AI-generated content can be wrong. Capability-framing standards (HAX G1/G2) require
a visible disclaimer near AI output so users know to verify critical information before
acting on it. GitLab Pajamas defines a canonical disclaimer string that represents
the highest-confidence implementation of this requirement.

This rule detects whether a disclaimer is co-located with an AI-marker component.
Marker without disclaimer emits `warning`. Disclaimer present emits `info`.

## How

Globs `**/*.{tsx,jsx,vue}`, reads each file, and applies two detectors:

1. **Disclaimer detector** ŌĆö matches disclaimer text phrases (word-boundary, case-insensitive)
   or `*Disclaimer*` / `*AIDisclaimer*` component tags; flags the GitLab Pajamas
   exact canonical wording as highest-confidence.
   Detected phrases: `Generated by AI`, `AI-generated`, `May be inaccurate`, `Check important info`,
   `Powered by AI`, `AI-assisted`, `Uses AI. Verify results.`, `Verify results / before / the output`.
2. **AI-marker detector** ŌĆö reuses `AI_MARKER_NAMES` and `isAiMarkerName` from
   `ai-governance/ai-marker-component-present` to recognise Carbon `AILabel`, `AIBadge`,
   `AITag`, Polaris `magic-*`, `GenAI*` variants, and `*AIMarker*` components.

Cross-condition: marker present + no disclaimer emits one **aggregate** `warning` listing all
affected files (up to 20, then "+N more"). Disclaimer detected emits `info` per file.
No AI surface emits nothing.

## Examples

**Good** ŌĆö marker + disclaimer:

```tsx
<AILabel />
<p>{summary}</p>
<p className="disclaimer">Generated by AI. May be inaccurate.</p>
```

**Good** ŌĆö new phrasings (all detected):

```tsx
{/* AI-assisted variant */}
<span>AI-assisted</span>

{/* Uses AI + verify pattern */}
<p>Uses AI. Verify results.</p>

{/* Verify results/before style */}
<p>AI output ŌĆö verify results before acting.</p>
```

**Better** ŌĆö GitLab Pajamas canonical wording (highest-confidence):

```tsx
<AIDisclaimer message="AI-generated content may be inaccurate. Always check important information." />
```

**Bad** ŌĆö marker with no disclaimer:

```tsx
<AILabel />
<p>{summary}</p>
{/* no disclaimer rendered */}
```

## Auto-fix

None. Disclaimer wording is product-specific; the rule surfaces the gap.

## Allowlist

Add to `README.md` or `.lyse.yaml` in the repo root:

```
lyse-disable ai-governance/disclaimer-present
```

Use when the design system intentionally defers disclaimer rendering to the consumer app
and documents this contract explicitly.
